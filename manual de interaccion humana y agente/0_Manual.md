# 📘 Manual de Interacción Humana y Agente (ISP v2.3)

Este documento define el **Ritual de Colaboración** para asegurar que el Indra Satellite Protocol opere sin entropía.

## 🏛️ El Axioma de la Verdad
La única fuente de verdad es el **Contrato de Resonancia** (`indra_contract.json`). Ni el Humano ni la IA deben proponer cambios que violen este ADN sin antes actualizarlo formalmente.

---

## 🚀 El Ritual de Ignición (Para el Agente IA)

Antes de realizar cualquier tarea de desarrollo o automatización, el Agente debe seguir estos pasos en orden estricto:

### 1. Auditoría del Nexo
Ejecutar `bridge.audit()` en la terminal o consola para verificar:
- [ ] ¿El Satélite está sincronizado con el ADN local?
- [ ] ¿Los protocolos requeridos están asimilados?
- [ ] ¿Existe una conexión válida (o modo local activo)?

### 2. Validación Basada en HUD (Evidence-Based Design)
El Agente no entrega código "ciego". Utiliza la interfaz del **IndraBridgeHUD** para:
- Visualizar que el esquema de datos aparezca en el **Árbol de Resonancia**.
- Confirmar que el **Workspace** activo es el correcto para la tarea.
- Verificar que el **Workflow Designer** proyecte la lógica de automatización deseada.

### 3. La Regla de la Única Iteración
El objetivo del Agente es que el Humano solo tenga que refrescar el navegador una vez. Si el Agente ha validado su trabajo contra el HUD, el resultado debe ser funcional, estético y soberano de inmediato.

---

## 🛡️ Protocolo de Seguridad y Agnosticismo
- **Sin Nombres Propios**: Prohibido hardcodear nombres de empresas, correos o IDs de producción en la carpeta `_INDRA_PROTOCOL_`.
- **Soberanía de Silos**: El Agente debe proponer siempre el uso de `silo_mapping` para que el satélite sea independiente del proveedor de datos (Sheets, Notion, etc).

---

## 💎 El Dharma de la Entrega
Cada tarea finalizada debe terminar con un **Resumen de Resonancia**, indicando qué cambió en el contrato y cómo se visualiza esa mejora en el HUD.
